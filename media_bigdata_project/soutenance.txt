1. Data Sources — Collecte (Web Scraping)
Ce qu'on a fait : Un scraper Python (scraper_hespress.py) qui collecte automatiquement des articles depuis BBC News et Reuters en parsant le HTML avec BeautifulSoup. Il extrait le titre, l'URL, la source et la date de collecte, puis sauvegarde tout en JSON dans le dossier data/local/bronze/.
Ce que tu dis au prof : "On a développé un scraper Python qui collecte automatiquement des articles de presse toutes les heures depuis plusieurs sources internationales. Les données brutes sont sauvegardées en JSON dans la couche Bronze du Data Lake."

2. Ingestion — Double mode Batch + Streaming
Ce qu'on a fait : Deux modes d'ingestion coexistent :

Batch : le scraper tourne toutes les heures (orchestré par Airflow)
Streaming : streaming_producer.py envoie chaque article comme événement vers Apache Kafka en temps réel

Ce que tu dis au prof : "On a implémenté deux modes d'ingestion : le batch toutes les heures via Airflow, et le streaming temps réel via Apache Kafka qui reçoit chaque article comme un événement indépendant."

3. Data Lake — MinIO (Architecture Médaillon)
Ce qu'on a fait : On a déployé MinIO (équivalent S3 en local) comme Data Lake avec un bucket news-datalake. Les données sont organisées en 3 couches Médaillon :

Bronze : données brutes JSON telles quelles sortent du scraper
Silver : données nettoyées — suppression HTML, normalisation du texte, détection de langue, déduplication, filtrage des articles trop courts
Gold : données agrégées et enrichies — top mots-clés, articles par source, par jour, par catégorie

Ce que tu dis au prof : "Le Data Lake est basé sur MinIO. On a implémenté l'architecture Médaillon avec 3 couches : Bronze pour les données brutes, Silver pour les données nettoyées et normalisées, et Gold pour les agrégations analytiques prêtes à l'analyse."

4. Transformation ETL — Python
Ce qu'on a fait : Deux scripts de transformation :

bronze_to_silver.py : nettoie le HTML, normalise le texte, détecte la langue avec langdetect, filtre les doublons par URL
silver_to_gold.py : calcule les mots-clés avec Counter, produit les CSV analytiques (tendances, top sujets, volumes)

Ce que tu dis au prof : "Les transformations ETL sont réalisées en Python pur. La pipeline Bronze→Silver nettoie et normalise les données, la pipeline Silver→Gold les enrichit et les agrège pour produire les tables analytiques."

5. Data Warehouse — PostgreSQL
Ce qu'on a fait : On a déployé PostgreSQL comme Data Warehouse avec 4 tables analytiques créées automatiquement :

articles_gold — tous les articles enrichis
articles_par_source — volume par média
articles_par_jour — tendance temporelle
top_mots_cles — mots-clés les plus fréquents

Ce que tu dis au prof : "Le Data Warehouse PostgreSQL stocke les données Gold sous forme de tables analytiques optimisées pour les requêtes décisionnelles et la visualisation."

6. Orchestration — Apache Airflow
Ce qu'on a fait : Un DAG Airflow (news_bigdata_pipeline) orchestre tout le pipeline automatiquement toutes les heures dans cet ordre :
Scraping → Bronze→Silver → Qualité → Silver→Gold → Chargement PostgreSQL
Ce que tu dis au prof : "Apache Airflow orchestre et planifie automatiquement toutes les étapes du pipeline. Le DAG s'exécute toutes les heures, supervise chaque tâche et gère les dépendances entre elles."

7. Qualité des données
Ce qu'on a fait : data_quality_checks.py génère un rapport JSON qui vérifie :

Articles sans titre
Dates manquantes
Contenu trop court (< 20 mots)
URLs manquantes
Taux de complétude global

Ce que tu dis au prof : "On a mis en place des contrôles de qualité automatiques qui vérifient la complétude, la cohérence et la validité des données à chaque exécution du pipeline."

8. Visualisation — Metabase
Ce qu'on a fait : Metabase est connecté à PostgreSQL et détecte automatiquement les 4 tables analytiques. On peut créer des dashboards pour visualiser les tendances, les volumes par source et les mots-clés.
Ce que tu dis au prof : "Metabase est connecté au Data Warehouse et permet de créer des dashboards interactifs pour visualiser les tendances d'actualité, les volumes d'articles par source et les mots-clés dominants."

9. Déploiement — Docker
Ce qu'on a fait : Toute l'architecture est containerisée avec Docker Compose en un seul fichier docker-compose.yml qui lance 6 services : MinIO, Zookeeper, Kafka, PostgreSQL, Airflow, Metabase.
Ce que tu dis au prof : "Tout l'environnement est déployé avec Docker Compose. Un seul fichier suffit à lancer l'intégralité de l'architecture — Data Lake, broker de messages, Data Warehouse, orchestrateur et outil de visualisation — de manière reproductible."
